# Assets Architecture

## Purpose

The assets folder stores shared non-code project assets such as brand material, images, videos, icons, wireframes, sample documents, and exported design artifacts.

## Folder Structure

```text
assets/
  brand/                  Logos, colors, typography, and brand guidelines
  images/                 Shared image assets
  icons/                  Shared icon assets
  videos/                 Demo and product media
  documents/              Safe sample documents and templates
  wireframes/             Design wireframes and exported UX artifacts
  exports/                Generated design or reporting exports
```

## What Belongs Here

- Brand logos and guidelines
- Product images and placeholders
- Icons and visual assets
- Demo videos
- Synthetic sample resumes, SOPs, and reports
- Wireframes and design exports

## Best Practices

- Do not store private student documents here.
- Prefer optimized assets for production use.
- Keep source design files clearly named and versioned.
- Separate shared assets from frontend implementation assets when needed.
